# Emission intercomparison V2 Stage-9 frozen setup

Status: implementation and deployment kit complete; no Stage-9 science,
preflight, pilot, injection, or retrieval has been executed.

## Science boundary

Stage 9 asks how native ROBERT, PICASO, and petitRADTRANS emission-model
differences propagate into atmospheric and grey-cloud posterior inference.
Only Track B is used. Each framework owns its native opacity mapping and RT
solver; no common gas/cloud tensor or invented solver mode is passed between
frameworks.

The four scenarios are clear PG14 non-inverted, clear PG14 inverted, grey
absorbing PG14 non-inverted, and grey isotropically scattering PG14 non-
inverted. The controlled Stage-8 result justifies the last scenario only at
grey extinction, zero asymmetry, no delta-M, and the accepted moderate cloud
placement. MgSiO3, Mie optics, wavelength-dependent extinction, anisotropy,
microphysics, and high-order scattering remain a future study.

The clear cases retrieve five PG14 coordinates and log10 VMRs for H2O, CO,
CO2, and CH4. `T_int=100 K` and the common physical radii/gravity/eclipse
normalization are fixed; no area-scale coordinate exists. The absorbing cloud
adds log10 optical depth at 5 micron and log10 cloud-top pressure. The
scattering cloud adds native single-scattering albedo. Cloud bottom is 100 bar,
slope is zero, asymmetry is zero, and delta-M is off.

## Matrix and sampler

There are six directed cross-framework pairs at 30, 60, and 100 ppm. Every
cell has one noiseless mean and five deterministic Gaussian-noise seeds.
Self-retrieval controls use only 60 ppm with the same six realizations. This is
108 cross plus 18 self retrievals per scenario, 126 per scenario, and 504 in
total. The five noisy realizations support descriptive `x/5` coverage counts,
not precision calibration claims.

Every retrieval uses MultiNest 3.10/PyMultiNest 2.12 with 400 live points,
evidence tolerance 0.5, sampling efficiency 0.8, importance nested sampling
and multimodality enabled, update interval 100, unlimited iterations, exact
resume, and invalid likelihood floor `-1e100`. A retrieval uses 12 MPI ranks
and one numerical-library thread per rank. Glamdring `addqueue` starts those
ranks; nested `mpirun` or `srun` is forbidden.

## Files and operation

The immutable science contract is
`docs/data/emission_intercomparison/version_2/stage_9_retrieval_contract.json`.
`scripts/prepare_emission_intercomparison_v2_stage_9.py PROJECT_ROOT` creates
all 504 run directories, twelve 42-run shard files, twelve injection slots,
60 shared standard-normal vectors, cache/reference/archive/integrity trees,
and exact run JSON files. Re-running is idempotent; changed definitions fail.

Install the three Glamdring environments with:

```bash
scripts/install_emission_intercomparison_v2_stage_9_glamdring.sh \
  /path/to/ROBERT-code /path/to/conda-prefixes
```

The PICASO and pRT definitions are
`environment-stage9-picaso-v4.yml` and
`environment-stage9-petitradtrans.yml`. Shared reference trees are staged with
`scripts/stage_emission_intercomparison_v2_stage_9_reference_data.py`; symlink
mode avoids opacity duplication and writes a per-file SHA-256 inventory.

After explicit approval, each of the twelve native means is generated in its
framework environment with
`examples/generate_emission_intercomparison_v2_stage_9_injection.py`. One
retrieval is run by
`examples/run_emission_intercomparison_v2_stage_9_retrieval.py`; production
shards are submitted by
`scripts/queue_emission_intercomparison_v2_stage_9_shard.sh`. Failed jobs keep
their native MultiNest restart files. Successful jobs retain compressed
samples/weights/log likelihood, evidence, summaries, and only best-fit plus
posterior-median spectra. Raw chains can be archived and removed only after
`scripts/archive_emission_intercomparison_v2_stage_9_run.py` reopens and
checksums the archive.

Posterior overlays and five-seed coverage products are generated by
`examples/plot_emission_intercomparison_v2_stage_9_posteriors.py`. Native
injection comparisons and best-fit/posterior-median spectral residuals are
generated by `examples/plot_emission_intercomparison_v2_stage_9_spectra.py`.

## Preliminary resources and storage

Twelve shards are keyed by retriever and scenario. Every shard contains 42
sequentially submitted 12-rank jobs. Provisional RAM is 96 GB for clear ROBERT,
128 GB for cloudy ROBERT, 32 GB for PICASO, and 64 GB for pRT, with a provisional
48-hour job wall limit. These are pilot inputs, not measurements.

Storage is bounded by storing 12 injection means and 60 noise vectors once,
using shared reference trees, never retaining per-likelihood or full posterior-
predictive spectra, compressing canonical posterior arrays, and archiving raw
MultiNest chains after validation. The working target is 150--250 GB and the
archive target 100--250 GB; any pilot projection above 500 GB is a no-go and
must be redesigned before production.

## Approved preflight/pilot sequence

No step below runs until the user explicitly approves it on Glamdring.

1. Verify all environment versions, MPI ABI/world size, MultiNest loading,
   reference paths/checksums, scratch writeability, common/frozen contract
   hashes, and exact 504/12/42 matrix counts without evaluating a spectrum.
2. Generate one native truth spectrum per framework for clear non-inverted and
   grey scattering (six forward calls total); compare grids, finiteness, and
   retained sizes.
3. Time cold and warm truth evaluations for those six cells, recording full
   12-process-tree RSS and scratch growth with
   `examples/pilot_emission_intercomparison_v2_stage_9_forward.py`.
4. Run one short, explicitly non-production MultiNest resume/checkpoint pilot
   per framework, then interrupt and resume it. Pilot outputs use separate
   directories and can never satisfy a production run manifest.
5. Recalculate RAM, wall-time, and storage projections. Freeze requests only
   if all gates pass.

## Go/no-go checklist

Production is GO only when environment and reference checksums match; all 12
injections are finite and use the common R=100 grid; paired noise vectors are
identical by ID; 12-rank MPI and resume work for all frameworks; no run exposes
area scale or out-of-scope cloud parameters; resource projections fit the
selected queue with at least 25% RAM and wall-time margin; projected working
storage is at most 500 GB; failed runs remain visible and resumable; and the
user explicitly authorizes production submission. Any missing native
capability, fabricated shared tensor, environment/ABI mismatch, checksum
drift, non-finite spectrum, resume failure, or resource/storage gate failure
is NO-GO.

The bounded pilot interface is the normal retrieval runner with
`--pilot-output PROJECT_ROOT/pilots/FRAMEWORK/CASE`. It defaults to 50 live
points and 200 iterations, changes the problem name, refuses any output below
the production `runs/` tree, and writes a `PILOT_ONLY` marker. Repeating the
same command exercises MultiNest resume; production settings and directories
remain untouched.
